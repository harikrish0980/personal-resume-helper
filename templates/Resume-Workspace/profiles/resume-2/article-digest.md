# Resume 2 Article Digest

Use this file as the proof bank for Resume 2. Add only real experience points that you can explain in an interview. The app should select from these points when they match the job description; it should not invent unrelated achievements.

## How To Add Experience Points

Use client-wise, company-wise, or project-wise sections. Keep each bullet specific: action, tool, scope, result, and metric if true.

### Client Or Company: Example Client Name
Role: Example Role
Dates: Month YYYY - Month YYYY
Domain: Banking, healthcare, retail, telecom, SaaS, or other real domain
Tools: Python, SQL, Spark, Databricks, Snowflake, Azure, AWS, Power BI, Tableau, etc.

- Built [real system/process/report] using [tools] for [team/client], resulting in [real outcome or metric].
- Improved [performance/reliability/quality/process] by [real action], reducing [time/cost/errors] by [real metric if known].
- Partnered with [stakeholders/team] to solve [business problem], delivering [real result].
- Migrated, automated, supported, or documented [real work] with [tools/platforms] and [impact].

### Project: Example Project Name
Problem: What business or technical problem did this solve?
Your contribution: What did you personally do?
Outcome: What changed after your work?

- Add a strong, factual bullet here.
- Add another bullet only if it is different and useful for job matching.

## Rules

- Do not add fake metrics, fake clients, or fake responsibilities.
- Do not paste a whole resume here; use this as extra proof points and project detail.
- Prefer concise bullets that can be reused in a resume.
- Keep cloud/tool claims consistent with the selected resume profile.
